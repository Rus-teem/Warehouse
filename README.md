# Warehouse

Набор небольших скриптов, шаблонов и служебных файлов для разных задач: парсинг, автоматизация, HTML/CSS/JS-черновики, Docker и SQL.

## Что где лежит

- `ad/` - текстовые файлы с рекламными и площадочными данными.
- `CSS/` - отдельные CSS-стили.
- `Docker/` - Docker-приложения и окружения.
- `GTM/` - HTML-шаблоны для целей/событий и отправки данных.
- `HTML/` - HTML-файлы, в том числе structured data.
- `JS/` - JavaScript-скрипты и связанные HTML/CSS-файлы.
- `Parsing_results/` - результаты парсинга в `csv` и `xlsx`.
- `Python/` - Python-скрипты для парсинга, WhatsApp, анализа и тестов.
- `SQL/` - SQL-запросы.
- `User_Data/` - локальные пользовательские/браузерные данные.
- `Warehouse/` - вложенная папка проекта с отдельными файлами.

## Основные файлы

- `README.md` - описание проекта.
- `.gitignore` - исключения для Git.
- `LICENSE` - лицензия.
- `qodana.yaml` - настройки Qodana.
- `automation_combined_log.txt`, `browser_actions.log`, `dom_mutations.json` - служебные логи и дампы.
- `whatsapp_web_dump.html` - сохраненная HTML-страница.
- `meeting.ics` - календарное событие.

## Примечание

В проекте есть временные и сгенерированные файлы, поэтому содержимое папок может меняться по мере работы.
